# 前端与联调待开发计划文档

本文档基于对当前代码库现状的分析，梳理了前端（含小程序、Web C端、Web 管理端）需要落地的真实业务逻辑开发任务，以及存在堵塞情况需要后端配合补齐的 API 事项。

## 一、小程序端 (Mini Program)

### 1. 基础架构与业务落地
- **当前状态**：未开发。仅有占位说明 (`apps/miniprogram/README.md`)，无任何可运行代码。
- **研发任务**：
  - [ ] 确定技术栈模型（原生开发 / Taro / Uni-app）并搭建基础脚手架。
  - [ ] 落地首期核心链路页面：首页、匹配页、任务大厅、个人中心。
  - [ ] 接入微信生态能力：微信快捷登录、获取手机号授权、订阅消息统一通知机制等。

## 二、Web 用户端 (Web - Client)

### 1. 团长中心 (`apps/web/app/captain/page.tsx`)
- **当前状态**：纯 Mock 数据驱动。邀请链接、佣金流水列表、统计数字均为本地硬编码。
- **研发任务**：
  - [ ] 联调 `/captain/info` 接口，替换真实的团长等级、专属邀请海报与邀请链接获取。
  - [ ] 联调 `/captain/stats` 接口，展示真实的推广拉新数据分析。
  - [ ] 联调 `/captain/commissions` 接口，动态渲染佣金到账流水并实现提现能力。

### 2. 会员开通与支付 (`apps/web/app/member/page.tsx`)
- **当前状态**：没有走真实的后端支付链路。目前前端仅用 `sleep` 本地模拟进度，并调用状态管理强制升级（强制提升 MemberLevel）。
- **研发任务**：
  - [ ] 请求真实的会员售卖层方案列表 API。
  - [ ] 调用真实的收银台服务下单（如微信/支付宝拉起）。
  - [ ] 前端实现支付结果轮询等待或完成重定向，真实拉取最新的权限 Level 保存至本地 Store。

### 3. 个人中心 (`apps/web/app/profile/page.tsx`)
- **当前状态**：实名认证与头像上传属前端单机模拟（未经过后端业务验证与存储验证）。
- **研发任务**：
  - [ ] 对接云存储系统（OSS等），改造本地 DataURL 形式的头像上传，换成直传并将其存入用户身份数据库。
  - [ ] 对接真实的中国大陆身份库校验接口（目前直接兜底为 `success: true`）。

## 三、Web 管理端 (Web - Admin)

### 1. 首页控制台 (`apps/admin/app/page.tsx`)
- **当前状态**：当前并未发起真实的统计数据 API，`stats` 固定使用空数据源。
- **研发任务**：
  - [ ] 调试并接入真正的 `/admin/stats`（平台概览数据全貌），实现总用户量、日活、资源数及转化率大盘指标可视。

### 2. 运营活动监控 (`apps/admin/app/activity/page.tsx`)
- **当前状态**：未使用对应 API 获取真实赛榜，采用 `setTimeout` 加上测试排名进行模拟刷新。
- **研发任务**：
  - [ ] 调用真实的 `captainRanking` API 拉取船长拉新冲榜记录，替换硬编码赛区。
  - [ ] 增加局部轮询或 WebSocket 以支持在活动页面中实时看到冲榜数据的刷新效果。

### 3. 任务与审核 (`apps/admin/app/tasks/page.tsx`)
- **当前状态**：悬赏任务下发和审核列表为脱机 Mock。
- **研发任务**：
  - [ ] 接入后端对应的查询任务发布接口 `/admin/tasks`。
  - [ ] 联调对接审核结果接口（`/admin/submissions/:id/review`），实现审核审批流流转。

### 4. 平台公告投递 (`apps/admin/app/announcements/page.tsx`)
- **当前状态**：本地维持一个乐观表结构，发布即进入内存中显示，刷新立即丢失。且没有初始化获取后端历史记录。
- **研发任务**：
  - [ ] 首屏挂载时完整接入“拉取全量/分页公告”接口赋值至状态。
  - [ ] 发送广播时确保完成服务器真实落库动作后更新视图。

## 四、后端待补齐或预联调卡点

由于前端以上功能的接轨高度依赖后端的健壮度，开发前须与后端确认下述项已完成或排期：

1. **Schema 模型未落库**：报告核心所需的 `Opportunity` / `LobsterStatus` / `HpLog` / `Referral` 等不在当前 Schema 内，必须先完成数据架构与 `prisma db push`。
2. **AI 与自动化业务伪实现**：要求 AI 匹配（非现存伪随机模拟），业务中核心层须接入 DeerFlow / OpenClaw 等工作流的真实调度网关，内容生成也是纯模拟实现，需要接入 LLM 生成。
3. **接口路由漏配**：部分实现好的验证、发奖、购买、后台 Match 功能虽然有 Service 层方法但缺失 Controller 出口暴露，导致前端无路可调，需要后端同步将接口暴露并定义好 Swagger/TS 类型。
4. **第三方真实服务基建**：验证码此时还在内存 Map 并返回明文，实名验证仅是 Stub 实现。这些需挂载真实的通信商和政务接口环境后才能闭环联调。
