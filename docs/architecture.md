# 系统架构（基于需求图）

## 分层设计

1. 用户端：微信小程序 / H5 Web / 管理后台
2. 网关层：Nginx + API Gateway + JWT + 限流
3. 应用层：用户、资源、匹配、内容、团长、会员、通知
4. AI 引擎：小龙虾调度器、标签匹配、内容生成、合规审核
5. 数据层：MySQL、Redis、RabbitMQ、Elasticsearch、OSS
6. 基础设施：Docker + K8s + CI/CD + 云监控

## 模块与阶段

- Phase 1：用户注册认证、资源上传标签、智能匹配
- Phase 2：AI内容生产分发
- Phase 3：团长裂变 + 会员支付
- Phase 4：AI自主调度
- Phase 4-5：赛博分身系统

## 合规底线

- 不碰隐私：手机号仅哈希、实名只校验不落库
- 不碰钱：平台不存用户资金，仅收工具服务费
- 不碰违规：上传前、生成前、发布前多层审核
